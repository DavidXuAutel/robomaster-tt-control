0. 一句话概括

ScoutXWAM 是在紧凑型「世界-动作模型」ScoutWAM（2.06B 参数、LIBERO 平均成功率 91.1%）基础上做的下一代扩展，核心目标是把模型从"benchmark 上会预测动作"推进到"在想象空间里具备空间智能、并能把动作能力迁移到真实机器人"。技术手段：几何先验引导的世界建模 + 多视角 RGB-D + 轻量深度适配分支 + 动作/视频异步噪声调度 + 三层数据金字塔课程。

1. 立项动机

已有基础 ScoutWAM 是一个紧凑型世界-动作模型（World-Action Model）：

91.1%LIBERO（Spatial/Object/Goal/LIBERO-10）平均成功率
2.06B参数量，对比基线 Fast-WAM 的 6.70B
1,822/ 2,000 评测 episode 成功 rollout
52.0rollouts / GPU-hour（两卡评测）
4.21×LIBERO-10 聚合 rollout 吞吐（对比基线）

核心论点：仿真/benchmark 成功是必要但不充分的，下一步要验证学到的是可迁移的世界表征。为此 ScoutXWAM 提出两个待验证的核心问题：

编号问题验证内容
Q1世界模型能否在想象空间发展出空间智能？想象未来是否保持真实几何、物体关系、运动、可供性与动作的物理后果，而不仅是"看起来像"。
Q2动作性能能否迁移到真实世界？LIBERO 上的动作能力在真机（真实感知、本体、动力学、环境变化）下是否成立。

2. 架构设计与单步去噪流程

在 ScoutWAM 紧凑基础上扩展四点：几何先验引导的世界建模、成对多视角 RGB-D 观测、轻量深度适配分支、动作-视频异步噪声调度。输出直接嵌入高分辨率的 3D 重建 / 4D 重建 / 场景流 / 动作轨迹。

单步去噪会联合去噪想象视频 + 机器人状态 + 动作轨迹，同时辅助深度分支把共享表征拉向一致 3D 几何。

  • 输入：带噪视频隐变量 z_O^{tO}、状态 z_s^{ta}、动作 z_a^{ta}，加上初始观测 O_0、初始状态 s_0、语言指令 c。
  • 输出：视频/状态/动作速度场 v̂_O, v̂_s, v̂_a 与逆深度 D̂。
01

编码条件

Causal VAE 编码初始 RGB 观测；MLP 编码初始机器人状态。时刻 0 的干净信号作为 rollout 锚点。

02

组装隐变量序列

条件与带噪视频/状态/动作隐变量拼接，加可学习视角嵌入。独立的 t_O 与 t_a 让视频和动作走异步噪声调度。

03

构建共享上下文

序列经过 N − M 个共享 DiT 块，融合视觉、状态、动作、指令、相机视角上下文。

04

动力学与深度交织

主分支与深度分支从同一共享特征出发；最后 M 层深度单向 attend 主分支输入，主动力学流独立推进。

05

解码预测

主头预测视频/状态/动作去噪速度；轻量深度头单独回归逆深度，提供几何学习信号。

核心思想

主分支学习世界与动作动力学，单向深度适配器把共享隐空间引导向几何收敛——在不让动作生成依赖深度反馈的前提下，为表征补充空间结构。

4. DROID-1K Spatial 数据集

目标：在 4D 想象空间 rollout 学习空间智能——在三维几何、时间、视角、动作上联合推理。做法：用深度引导隐空间走向几何收敛，鼓励想象状态收敛到一致物理结构而非只学外观。

样本把机器人视觉观测与对应深度重建配对；声称达成稳定视频-深度估计、全 episode RGB 与深度流同步。

7. 零样本定性结果

设置：在 DROID-1K Spatial 训练，在训练从未包含的 Scout-GELLO 上零样本评测，检验空间表征迁移能力。迁移路径：DROID-1K Spatial → Scout-GELLO。

关键结论

预训练初始化 + DROID-1K 微调，产出强通用空间智能模型，能做准确的 3D 重建、4D 时空推理与场景流理解。展示含 RGB→深度序列，以及 3D/4D 重建与场景流三段零样本视频。

8. 具身特定后训练

目标平台 Franka。用 Scout-GELLO 微调通用 ScoutXWAM，对齐目标本体的观测、运动学、控制接口与真实工况，同时保留预训练 + DROID-1K 的广泛空间智能。

路线：Scout-GELLO 适配数据 → 具身对齐（微调感知/本体感知/动作到 Franka 控制空间）→ Franka 闭环部署测试。原页面中最终曲线、收敛分析、定性结果、真机部署结果均标注为 forthcoming（尚未公布）。

10. 对本项目的启示与可复用点

结合本仓"真机优先：真机飞 → 仿真记录轨迹 → 证明真机↔仿真匹配 → 部署'别人的模型'跑真机"的思路：

  1. 深度作为几何先验可对接避障。 本项目 tt_control/depth_backend.py / avoidance.py 已有"深度图 → 最近障碍方位 → 避让杆量"设想；若接入的模型是世界/深度类模型，可对齐 RGB-D、逆深度输入与速度场/动作输出约定。
  2. 异步动作-视频噪声调度 = 世界预测与动作生成解耦。 设计 policy.py 的 ExternalModelPolicy adapter 时，注意动作输出可能不与视频预测同步。
  3. real-to-sim-to-real 闭环与本项目链路同构。 Scout-GELLO 的"真采集→仿真扩充→回流真机"正是本项目链路的通用化版本，可作数据采集/回归验证模板。
  4. 零样本迁移评测方法可借鉴。 "A 分布训练、未见 B 分布评测重建/动作质量"是判断表征可迁移性的实用手段；真机↔仿真匹配验证可用类似留出集 + PSNR/SSIM/MAE/RMSE 指标量化。
  5. 接口约定备忘。 输入：初始 RGB 观测、初始机器人状态、语言指令、（可选）多视角与深度；输出：动作序列/速度场（Δx·Δy·Δz·ΔR·G）与可选深度/重建。TT 无人机把 Δx·Δy·Δz·ΔR 映射到 RcAxes 杆量即可复用现有下发通路。
