## 0. 一句话概括

ScoutXWAM 是在紧凑型「世界-动作模型」ScoutWAM（2.06B 参数、LIBERO 平均成功率 91.1%）基础上做的**下一代扩展**，核心目标是把模型从"benchmark 上会预测动作"推进到"在想象空间里具备**空间智能（spatial intelligence）**、并能把动作能力**迁移到真实机器人**"。技术手段可归纳为：**几何先验引导的世界建模 + 多视角 RGB-D + 轻量深度适配分支 + 动作/视频异步噪声调度 + 三层数据金字塔课程**。

---

## 1. 立项动机（Motivation）

- **已有基础 ScoutWAM**：紧凑型世界-动作模型（World-Action Model）。
  - 参数量 **2.06B**，对比最接近的 Fast-WAM 基线的 **6.70B**。
  - LIBERO 四个子集（Spatial / Object / Goal / LIBERO-10）**平均成功率 91.1%**。
  - 2,000 次评测 episode 中 **1,822 次成功 rollout**。
  - 最终两卡评测吞吐 **52.0 rollouts / GPU-hour**。
  - LIBERO-10 聚合 rollout 吞吐是基线的 **4.21×**。
  - 推理 PSNR **30.35 dB**（2.06B）对比 Fast-WAM 6.70B。
- **核心论点**：仿真/benchmark 成功是**必要但不充分**的。下一步要验证模型学到的是**可迁移的世界表征**，而不仅仅是一个在 benchmark 上高分的策略。

### 两个待验证的核心问题
| 编号 | 问题 | 验证内容 |
|---|---|---|
| Q1 | 世界模型能否在**想象空间**里发展出空间智能？ | 想象出的未来是否保持真实世界几何、物体关系、运动、可供性（affordance）与动作的物理后果，而不仅是"看起来像"。 |
| Q2 | 动作性能能否**有利地迁移到真实世界**？ | LIBERO 上的动作能力在真实机器人上（真实感知、真实本体、动力学、环境变化下）是否依然成立。 |

## 2. 架构设计（ScoutXWAM Architecture）

在 ScoutWAM 紧凑基础上扩展了四点：
1. **几何先验引导的世界建模**（geometric-prior-guided world modelling）
2. **成对多视角 RGB-D 观测**（paired multiview RGB-D）
3. **轻量深度适配分支**（lightweight depth adaptation）
4. **动作-视频异步噪声调度**（asynchronous action-video noise schedules）

**输出**：直接在架构的 Output 模块里嵌入高分辨率的 **3D 重建 / 4D 重建 / 场景流（scene flow）/ 动作轨迹**。

### 单步去噪流程（Single Denoising Step）

一次前向 pass 会**联合去噪**：想象视频 + 机器人状态 + 动作轨迹；同时一个**辅助深度分支**把共享表征拉向一致的 3D 几何。

- **输入**：带噪的视频隐变量 `z_O^{tO}`、状态隐变量 `z_s^{ta}`、动作隐变量 `z_a^{ta}`，加上初始观测 `O_0`、初始状态 `s_0`、语言指令 `c`。
- **输出（预测）**：视频/状态/动作的速度场 `v̂_O, v̂_s, v̂_a`，以及逆深度 `D̂`。

五个阶段：
1. **编码条件**：Causal VAE 编码初始 RGB 观测；MLP 编码初始机器人状态。这些"时刻 0"的干净信号作为 rollout 的锚点。
2. **组装隐变量序列**：将条件与带噪的视频/状态/动作隐变量拼接，并加上可学习的视角嵌入（view embeddings）。**独立的 `t_O` 与 `t_a`** 让视频和动作走**异步噪声调度**。
3. **构建共享上下文**：序列经过 `N − M` 个共享 DiT 块，融合视觉证据、机器人状态、动作、指令、相机视角上下文。
4. **动力学与深度交织**：主分支与深度分支从同一共享特征出发；在最后 `M` 层，深度分支单向 attend 主分支输入，主动力学流独立推进。
5. **解码预测**：主头预测视频/状态/动作的去噪速度；轻量深度头单独回归逆深度，提供几何学习信号。

> **核心思想（Core idea）**：主分支学习世界与动作动力学，一个**单向深度适配器**把共享隐空间引导向几何收敛——在不让动作生成依赖深度反馈的前提下，为表征补充空间结构。

## 3. 数据课程：三层数据金字塔（Closing the Data Gap）

从"广覆盖"到"多样真实交互"再到"任务特定演示"，逐层收敛：

| 层级 | 名称 | 定位 | 内容 |
|---|---|---|---|
| Level 03（顶） | **Scout-GELLO** | Apex · Specialise | 任务特定的真实世界演示，对齐 Scout 机器人与真实任务。 |
| Level 02（中） | **DROID-1K Spatial** | Middle · Ground | 多样真实机器人交互：不同场景、视角、物体、操作者，增强控制仿真外的鲁棒性。 |
| Level 01（底） | **5,800 小时** | Foundation · Scale | 真实机器人 + 仿真操作数据，覆盖多样任务、本体、环境与交互模式。 |

**基础数据集清单（Foundation datasets）**：
- AgiBot World Beta
- InternData-A1 · ALOHA
- InternData-A1 · Genie
- InternData-A1 · Lift2
- RoboCasa · MimicGen
- RoboTwin 2.0

## 4. 数据集构建之一：DROID-1K Spatial

**目标**：通过在 **4D 想象空间**中 rollout 来学习具备空间智能的世界模型——需要在三维几何、时间、视角、动作上联合推理，是本质上高维的学习问题。

**方法论**：
- **挑战**：有用的 rollout 必须保持的不只是视觉外观，还要维持物体几何、空间关系、运动、遮挡以及动作的物理后果随时间演化。
- **做法**：用**深度（depth）引导隐空间走向几何收敛**。这个几何先验鼓励想象状态收敛到一致的物理结构，而不是只学外观。

**数据集统计**：
| 指标 | 数值 |
|---|---|
| 机器人演示 episode | 1,000 |
| 同步时间步（timesteps） | 281,119 |
| 真实机器人演示时长 | 5.21 h |
| 单 episode 平均时长 | 18.74 s |
| 录制帧率（全部 3,000 路相机视频） | 15 FPS |

**样本展示要点**：每个样本把机器人的视觉观测与对应的深度重建配对，暴露用于构建想象空间的几何信号。声称达成：稳定的视频-深度估计；全 episode 上 RGB 与深度流同步。

## 5. 数据集构建之二：Scout-GELLO（real-to-sim-to-real 平台）

为支持模型后处理/精修，团队搭建了一个**自动化 real-to-sim-to-real 平台**，用于可扩展的数据采集，闭环连接：物理演示 → 仿真可用经验 → 部署回真机。

采集工作流（三步循环）：
1. **01 · Real 采集目标演示**：在物理 Scout 平台上采集高价值、具身特定的交互数据。
2. **02 · Sim 扩充经验**：把采集到的经验转成可重复、仿真可用的数据，以高效精修模型。
3. **03 · Real 回流精修**：把改进后的行为带回机器人做真实验证，进入下一轮采集。

Scout-GELLO 位于训练金字塔顶端，提供具身特定数据，让后处理聚焦到部署时真正重要的机器人、任务与工况上。数据样本以同步多模态回放呈现（动作轨迹 + 外部/腕部 RGB 观测 + 对应几何视图）。

## 6. 模型训练与验证

**算力与耗时**：
- 数据预处理：单张 **NVIDIA RTX 4090**，**48 小时**。
- DROID-1K Spatial 训练：**DGX Spark**，记录运行 **1.104 天**。
- 训练联合优化：视频、深度、机器人本体感知（proprioception）三个目标，共 **9,079** 个优化步。

**训练损失收敛（最终值）**：
| 目标 | 显示值 | 平滑值 |
|---|---|---|
| Video loss | 0.1824 | 0.2162 |
| Depth loss | 0.039 | 0.082 |
| Proprioception loss | 0.0107 | 0.0109 |

**留出集验证（Held-out，冻结 9K checkpoint，未见过的 DROID 演示）**：
| 输出 | 指标 | 均值 |
|---|---|---|
| Normalized action | MAE | 0.2646 |
| Normalized action | RMSE | 0.3492 |
| Normalized proprioception | MAE | 0.1407 |
| Normalized proprioception | RMSE | 0.3057 |
| RGB | PSNR | 16.61 dB |
| RGB | SSIM | 0.6734 |
| Normalized depth | PSNR | 20.97 dB |
| Normalized depth | SSIM | 0.8764 |

## 7. 定性结果：Scout-GELLO 上的零样本评测

- **设置**：模型在 **DROID-1K Spatial** 上训练，然后在**训练中从未包含**的 **Scout-GELLO** 演示上评测（零样本），检验学到的空间表征能否迁移到新的具身特定数据分布。
- **迁移路径**：训练分布 `DROID-1K Spatial` → 未见评测分布 `Scout-GELLO`。
- **关键结论**：预训练初始化 + DROID-1K 微调，产出一个强通用空间智能模型，能做准确的 **3D 重建、4D 时空推理、场景流理解**。
- 展示形式：RGB→深度序列（第三人称 + 腕部相机，首/中/末帧配对估计深度）；以及 3D 重建、4D 重建、场景流三段零样本视频。

## 8. 具身特定后训练（Embodiment-Specific Post-Training）

- **目标平台**：Franka 机器人。
- **做法**：用 **Scout-GELLO** 数据集微调通用 ScoutXWAM，对齐目标本体的观测、运动学、控制接口与真实工况，同时保留预训练 + DROID-1K 精修得到的广泛空间智能。
- **路线**：`Scout-GELLO 适配数据` → `具身对齐（微调感知/本体感知/动作预测到 Franka 控制空间）` → `Franka 部署（闭环测试动作质量）`。

## 10. 对本项目的启示与可复用点

结合本仓当前"真机优先：真机飞 → 仿真记录轨迹 → 证明真机↔仿真匹配 → 部署'别人的模型'跑真机"的思路，ScoutXWAM 的方案有几处值得借鉴/衔接：

1. **深度作为几何先验的思路可直接对接避障。**
   本项目 `tt_control/depth_backend.py` / `avoidance.py` 已有"深度图 → 最近障碍方位 → 避让杆量"的映射设想。ScoutXWAM 证明了"单向深度适配 + 几何收敛"是有效的表征引导方式；若未来接入的"别人的模型"是世界模型/深度类模型，可对齐这套输入（RGB-D、逆深度）与输出（速度场/动作）约定。

2. **异步动作-视频噪声调度 = 世界预测与动作生成解耦。**
   对"想象未来 + 生成动作"的场景，视频与动作走不同噪声时间轴，值得在设计 `policy.py` 的 `ExternalModelPolicy` adapter 时纳入考虑：动作输出可能不与视频预测同步。

3. **real-to-sim-to-real 闭环与本项目链路高度同构。**
   Scout-GELLO 的"真采集 → 仿真扩充 → 回流真机"三步循环，正是本项目"真机飞→仿真记录→部署真机"的通用化版本，可作为数据采集/回归验证流程的参考模板。

4. **零样本迁移评测方法可借鉴。**
   "在 A 分布训练、在从未见过的 B 分布上评测重建/动作质量"是判断表征是否可迁移的实用手段；本项目做真机↔仿真匹配验证时，可采用类似的留出集 + PSNR/SSIM/MAE/RMSE 指标体系来量化匹配度。

5. **接口约定备忘（供 `ExternalModelPolicy` 适配参考）。**
   若对接此类世界-动作模型，输入侧需要：初始 RGB 观测、初始机器人状态、语言指令、（可选）多视角与深度；输出侧为：动作序列/速度场（`Δx·Δy·Δz·ΔR·G`，即位置增量 + 姿态 + 夹爪）与可选的深度/重建。TT 无人机映射时把 `Δx·Δy·Δz·ΔR` 对应到 RcAxes 杆量即可复用现有下发通路。

## 附：术语与缩写
- **WAM**：World-Action Model，世界-动作模型。
- **DiT**：Diffusion Transformer，扩散 Transformer 块。
- **Causal VAE**：因果变分自编码器，用于视频编码。
- **LIBERO**：机器人操作任务 benchmark（含 Spatial / Object / Goal / LIBERO-10 子集）。
- **DROID**：大规模真实机器人操作数据语料。
- **GELLO**：低成本遥操作/数据采集方案（此处为 Scout-GELLO 平台）。
- **PSNR / SSIM**：峰值信噪比 / 结构相似性，图像重建质量指标。
- **MAE / RMSE**：平均绝对误差 / 均方根误差，动作与本体感知精度指标。
