# 大众具身智能机器人 · 遥操作数据飞轮系统 研究分析报告

> **文档定位**:本 MD 与同目录 HTML 报告内容同步,为纯文本版,供 Agent 检索、理解及作为任务背景注入。术语表见文末附录。

---

## 摘要(TL;DR)

大众做的是一套 **"人先遥操作干活、机器人边干边学"** 的具身智能系统:一只手控操作臂遥控远处的大型机械臂,移动操作臂即同步驱动真机与仿真中的机械臂,按操作臂按钮完成抓取/放手;整个过程同步采集多模态数据,形成**数据飞轮闭环**。

三个核心判断:

1. **架构主流且成立**:四层"快慢思考"分层 VLA(视觉-语言-动作)+ 遥操作数据飞轮,方向与当前具身智能主流范式一致,硬件已为数据采集预留能力,Demo 已跑通"遥操作—真机—仿真—采集"闭环。
2. **缺失的数据质量环是关键补齐项,且已在架构中被认知**:大众自述尚缺"低质量帧处理"与"任务完成情况质量评估";而后者("任务完成度评价")其实**已被列在架构图的快思考层**,只差落地。
3. **场景上"高危滩头 + 高频引擎"两条腿走**:以高压带电作业为首发滩头,核/化工跟进;同时用工业/物流等高频场景做数据飞轮引擎。无人机+机械臂技术难度大,宜作中期预研。

**最重要的结论**:该产品最大的差异化不在硬件,而在于**"遥操作即数据采集"的商业闭环** —— 选对一个"高危、高价值、动作可枚举"的作业场景做滩头,既能立刻用遥操作创造价值,又能在真实作业中滚出自主能力。

## 1. 原始方案解读

原始文档《结构及业务流.docx》正文仅含标题,核心信息由两张图承载:一张"整机软硬件架构 + 业务流 + 路线图"总览大图(`assets/fig_arch_full.jpg`),一张"硬件模组"分解图(`assets/fig_hardware.jpg`)。

### 1.1 硬件本体:轮式人形机器人

整机呈 **"头部 → 躯干 →(手臂 + 底盘)"** 连接结构,是一台带灵巧操作能力的轮式人形机器人。

| 模组 | 关键部件 | 作用 |
|---|---|---|
| **头部模组** | 麦克风阵列、深度相机(奥比中光/RealSense)、其它视觉传感器、显示模组、颈部电机组 | 多模态感知 + 人机交互(语音、视觉、表情/显示) |
| **躯干** | 核心算力板 **Thor**(NVIDIA Jetson Thor 级)、IMU、通信模组、数据采集板 + 电机驱动板、BMS+PACK、腰膝踝部电机组 | 算力与决策中枢、供电、下肢/腰部运动 |
| **手臂** | 关节模组、末端执行机构(灵巧手/夹爪)、深度相机 | 操作作业主体 —— 抓取/放置的执行末端 |
| **底盘** | 激光雷达、主控板(可选)、通信模组、深度相机、BMS+PACK、数据采集板 + 电机驱动板、轮系 | 移动、导航、SLAM 定位 |

> **要点**:硬件层面已为数据采集专门设计 —— **躯干与底盘都独立配置了"数据采集板 + 电机驱动板"**,视觉、力/位、语音等信号可在采集与控制回路中被完整记录,这是"边作业边采数据"的物理前提。

### 1.2 软件架构:四层"快慢思考"(分层 VLA)

采用经典的 **双系统(System-2 慢思考 / System-1 快思考)+ 控制 + 硬件** 四层结构:

- **① 慢思考(云端 Agent · System 2)** —— 云端大模型
  - 指令语义理解、环境理解、长程任务推理、长程任务(重)规划
- **② 快思考(端侧 Agent · System 1)**
  - **感知**:环境理解、传感感知、空间位置感知
  - **短程任务规划**:单一短程任务分解、短程任务规划/更新、**任务完成度评价**
  - **动作规划**:动作检测、动作执行、执行器状态
- **③ 控制(技能与运动)**
  - **技能库(操作技能)**:抓、放、推、拉
  - **技能库(运动技能)**:world plan、loco plan
  - **其他**:导航规划、关节角求解(IK)
- **④ 硬件层(传感 + 执行)**
  - **传感器**:相机、激光雷达、IMU、力矩传感器、RTK、麦克风
  - **执行器**:关节模组、柔性执行器、相机

> **关键发现**:大众自述缺失的"任务完成情况质量评估",在架构图里其实**已被明确列为快思考层的一个模块("任务完成度评价")** —— 说明团队已认知其必要性,只是尚未落地。第 4 章正是针对这一空白给出方案。

### 1.3 业务流:长程 → 短程 → 动作的任务分解

以生活化例子贯穿:用户说"渴了" → 云端大模型做**长程任务规划(LTP)** → 转化为"帮我拿瓶水" → 分解为若干**短程子任务**(移动到水前、抓一瓶水、返回递出) → 每个子任务再做**运动规划** → 下发控制信号驱动执行器。

```
文本/语音指令 → 长程任务规划 LTP(云端 LLM) → 短程子任务分解 → 运动规划
             → 技能库(抓/放/推/拉) → 控制信号 → 执行器
```

文档给出**两套技术路线权衡**:

- **Plan A · 大小脑分离**:"大脑"(云端)负责长程推理与自我迭代,"小脑/身体"(端侧)负责子任务规划、运动规划、控制。
  - ✔ 各司其职、云端可用更大模型　✘ 依赖网络、时延敏感
- **Plan B · 端侧统一模型**:长程规划、子任务规划、运动规划由一个端侧大模型统一承担。
  - ✔ 低时延、离线可用、适合危险/弱网现场　✘ 端侧算力与模型能力受限

### 1.4 系统拓扑:遥操作 + 仿真同步 + 数据采集(即 Demo)

这是所体验 Demo 的实现骨架,以 **ROS2** 为总线,把"人手遥操作 → 真机/仿真同步 → 多路数据采集"串成闭环:

- **遥操作侧**:手控操作臂 →「手臂运动板 / 手臂控制板 1、2」;移动操作臂产生位姿指令,按键触发夹爪抓/放。
- **控制中枢**:「总体控制系统核心板(ROS2)」接收遥操作指令,一路下发到**运动控制板 → 手臂关节 / 末端执行器**驱动真机同步动作;另一路送入**仿真系统(gym / Isaac 类)**,使远处大型机械臂与仿真中的机械臂同步运动。
- **数据采集侧**:「视觉数据采集系统 ×2」「声纹/麦克风数据采集」「全域数据采集板 + 数据处理」并行记录相机、深度、力/位、语音、IMU、LiDAR、RTK 等信号,写入**传感器数据存储**。
- **供电与移动**:BMS 供电;底盘由「导航控制核心板 2」结合 LiDAR + RTK 做导航,驱动轮系。

> **Demo = 数据引擎**:"移动操作臂 → 远处机械臂同步、按键抓放"其实是**数据采集终端** —— 人每完成一次抓放,系统就同步记录一条"多模态观测 + 动作"演示样本,并在仿真中留存镜像,为训练自主策略供料。

## 2. 数据飞轮闭环:为什么这套设计成立

核心逻辑:遥操作产生的每一条演示数据都用于训练自主策略,策略越强则需要人接管的比例越低,机器人越"自主",单位数据的作业产出越高 —— 形成正反馈。

```
① 人遥操作作业 → ② 采集多模态演示数据 → ③ 质检/清洗/打分
              → ④ 训练技能与 VLA 策略 → ⑤ 部署自主执行 ↺ ⑥ 失败处人接管纠偏
                                              └────(⑥ 回流到 ②③)────┘
```

- 第 ⑥ 步(人对失败场景的接管纠偏)回流到 ②③,是**价值密度最高的数据** —— 精准覆盖当前策略的能力边界。
- 飞轮转得好不好,取决于喂进去的**数据质量**,而不仅是数量(这正是第 4 章的重要性所在)。
- **仿真的作用**:真实遥操作轨迹可在仿真中做**域随机化 / 回放增广**,一条真实演示可放大为大量仿真样本(Real2Sim2Real),显著降低"堆真机数据"的成本。

## 3. 采集数据如何使用

遥操作采集的本质是成对的 **「多模态观测 → 动作」** 演示样本,可沿架构每一层被消费。

### 3.1 采集到的数据构成

| 类别 | 具体内容 |
|---|---|
| **视觉观测** | 多视角 RGB-D:环视相机、手臂腕部相机、头部相机(深度 + 彩色) |
| **本体状态** | 各关节角度/速度/力矩、末端执行器位姿、夹爪开合状态、IMU、底盘里程/RTK 位姿 |
| **动作标签** | 遥操作输入的末端 6-DoF 位姿轨迹、夹爪抓/放触发、底盘运动指令(即"人类演示的动作") |
| **语义/任务标签** | 指令文本/语音、子任务切分、任务成功与否(需第 4 章评估环节补齐) |
| **环境/其他** | LiDAR 点云、语音/声纹、时间戳与多传感器同步信息 |

### 3.2 数据的六种用法(对应架构各层)

1. **操作技能库训练(控制层)**:用演示数据以**模仿学习 / 行为克隆**训练"抓/放/推/拉"原子技能;主流方法如 **ACT(动作分块)、Diffusion Policy(扩散策略)**。最直接、最先见效。
2. **端侧 VLA 模型训练(快思考层)**:训练"视觉 + 语言 → 动作"的端侧大模型,遥操作数据即其监督训练集。
3. **短程规划与完成度评价**:子任务切分标签训练规划器;成功/失败标签训练"任务完成度评价"模块,让机器人自评"这一步做完没有"。
4. **世界模型(world plan)**:用"动作 → 未来观测"数据训练世界模型,支撑预测式规划与仿真推演。
5. **仿真协同(Sim Co-training)**:真实轨迹在仿真中回放 + 域随机化做 Sim2Real;也可 Real2Sim 建数字孪生,低成本扩充数据。
6. **云端规划蒸馏(慢思考层)**:长程任务分解的成功范例,用于微调/蒸馏云端规划大模型。

> **递进关系(投入顺序建议)**:短期靠 ①② 让单点技能可用 → 中期靠 ③④⑤ 提升泛化与自评、压低数据成本 → 长期靠 ⑥ + 飞轮跑通长程自主。不要一上来就追求端到端长程自主。

## 4. 数据质量处理方案(补齐大众所缺环节)

大众自述尚缺两块:**(A) 低质量帧的处理** 与 **(B) 任务完成情况的数据质量评估**。二者分别是**帧级**与**回合级(episode)**两个粒度的质检。

### 4.1 帧级:低质量帧的处理

**典型"坏帧"来源与检测手段:**

| 问题类型 | 表现 | 检测手段 |
|---|---|---|
| 图像质量差 | 运动模糊、失焦、过曝/欠曝、遮挡 | 拉普拉斯方差(清晰度)、曝光直方图、无参考图像质量评估(NR-IQA)模型 |
| 多传感器不同步 | 相机/关节/力信号时间戳错位 | 硬件时间戳对齐、跨模态同步校验、时间戳单调性检查 |
| 传输丢帧/卡顿 | 丢包、时延抖动、帧率不稳 | 时间戳间隔异常检测、帧序号连续性检查 |
| 运动异常 | 关节速度/加速度突变、越限、抖动(jerk) | 运动学阈值 + 平滑度指标、限位/碰撞检查 |
| 无效/停顿帧 | 遥操作犹豫、发呆、末端长时间不动 | 末端速度近零的时窗检测(空闲段裁剪) |
| 逻辑不一致 | 夹爪状态与接触力矛盾 | 夹爪状态 × 力/触觉一致性校验 |

**处理策略**:丢弃/遮罩坏帧;小间隙插值/重采样补齐并统一时钟;在坏帧断点处把演示**切成干净子轨迹**(而非整段作废);裁剪首尾/中段空闲段;高不确定样本转人工复核。

### 4.2 回合级:任务完成情况的数据质量评估

对每条完整演示(episode)判定**是否成功**并给出**质量分**,决定训练中的取舍与权重。

**自动成功判定(多信号融合):**
- **力 / 触觉**:抓取后是否维持稳定接触力 → 判定"抓稳没有"。
- **视觉核验**:目标物是否在夹爪内 / 是否到达目标位姿(检测 + 位姿匹配)。
- **VLM 裁判**:用视觉语言模型对"首帧指令 vs 末帧结果"打分,判定语义层面是否达成。

**质量打分维度:**

| 维度 | 说明 | 用途 |
|---|---|---|
| 成功与否 | 任务是否达成目标状态 | 正/负样本划分 |
| 轨迹效率 | 路径长度、耗时、能耗 | 过滤低效演示、加权 |
| 平滑/安全 | 抖动、碰撞、越界次数 | 剔除危险/低质演示 |
| 重试/犹豫 | 抓取重试、停顿次数 | 反映操作者水平 |
| 指令-行为一致性 | 动作是否匹配语言指令 | 保证语义对齐 |

**数据分级与取舍策略:**
- **金牌样本**(高质量成功)→ 主训练集,可上采样加权;
- **银牌样本**(低质量成功)→ 清洗后降权使用;
- **有价值失败**(信息量高的失败)→ 负样本 / RL 奖励信号 / 失败恢复数据;
- **废样本**(损坏/歧义)→ 丢弃或人工重标。

### 4.3 建议的数据质量流水线(7 步)

1. **采集与时间对齐** —— 按硬件时间戳对齐多传感器流,统一采样时钟,做同步性校验。
2. **帧级质检与清洗** —— 清晰度/曝光/丢帧/运动异常/空闲段检测 → 丢弃·插值·裁剪·切分(§4.1)。
3. **回合切分** —— 依子任务边界与坏帧断点,把长演示切为干净子轨迹。
4. **任务成功判定** —— 力/触觉 + 视觉核验 + VLM 裁判多信号融合,自动打成功/失败标签(§4.2)。
5. **质量打分与标签** —— 效率/平滑/一致性等维度计分,产出每条 episode 的质量分与标签。
6. **分级入库(金/银/铜)** —— 按质量分归档,构建可检索数据集与看板;抽样人工复核校准自动判定。
7. **加权/课程训练 → 失败回流** —— 按质量加权训练;模型评估暴露的失败场景优先安排遥操作补采,闭合飞轮。

> **落地建议**:把 §4 做成一个**独立的"数据质量服务"**(而非散落在采集脚本里):自动质检 + 人在环抽检 + 主动学习(优先复核不确定样本)。它直接对应架构图里已列出的"**任务完成度评价**"模块,是把飞轮从"能转"提升到"转得快"的关键组件,也是相对硬件更容易形成壁垒的软资产。

## 5. 应用场景分析

产品独特价值 = **遥操作(立即把人从危险/远距处撤出并创造作业价值)+ 数据飞轮(在作业中长出自主能力)**。
最佳场景 = **高危 · 高价值 · 人难以/不愿进入 · 但动作相对可枚举** 的作业。

### 5.1 场景全景

- **⚡ 电力带电作业**:高压线带电检修、绝缘子更换、异物清除。人在地面/车上遥操作,机器人上塔上线。高危、人力昂贵、需求刚性(电网)。
- **☢️ 核工业作业**:核电站检修、退役拆除、放射性废物处置、事故应急。辐射区人不宜久留,遥操作 + 自主是刚需。
- **🧪 化工 / 危化品 / 应急**:危化品搬运处置、泄漏堵漏、消防救援、爆炸物处置(EOD)。
- **🚁 无人机 + 机械臂(空中作业)**:由巡检升级为"空中动手":高压线带电修复、风机叶片维修、桥梁/大坝检修、高空清洗。
- **🌊 远程 / 无人环境**:海上平台、深海、矿山井下、灾害现场、乃至空间站舱外作业。
- **🏭 高价值工业 / 服务(飞轮引擎)**:柔性装配上下料、物流分拣、农业采摘、远程医疗辅助。高频、量大,最利于快速滚动数据飞轮。

### 5.2 对几个候选想法的专项评估

- **高压线带电作业 —— 最推荐的滩头场景**:危险替代价值极高、国内电网需求刚性且有政策支持、作业动作(剪、拧、清障、换绝缘子)相对可枚举、地面遥操作立刻可用。建议作为首个商业化落点。
- **核电危险作业 —— 高价值跟进场景**:付费能力强、"人撤出危险区"需求最刚性、容错要求高。适合在电力场景跑通后延伸,前期以遥操作为主、自主为辅。
- **无人机 + 机械臂空中作业 —— 差异化极高,但技术难度需正视**:核心挑战 ① 机械臂动作的反作用力扰动无人机姿态、悬停稳定性差;② 载荷与续航受限,大扭矩作业困难;③ 风扰下高精度对位难。建议路径:先用**大载荷多旋翼 / 栖停(perching)式或系留供电**平台,从"轻作业"(挂标、清障、喷涂、贴片检测)切入;空中带电重作业作为**中期研发赛道**而非首发场景。
- **AR/VR 眼镜遥操作 —— 是贯穿所有危险场景的交互增强,而非独立场景**:第一视角 + 力反馈能大幅降低遥操作门槛、提升采集数据质量,建议作为通用能力优先投入。

### 5.3 场景优先级矩阵

| 场景 | 危险替代价值 | 技术匹配度 | 数据飞轮友好度 | 落地难度 | 综合建议 |
|---|:--:|:--:|:--:|:--:|:--:|
| 电力带电作业 | ★★★★★ | ★★★★☆ | ★★★★☆ | 中 | **首发滩头** |
| 核工业作业 | ★★★★★ | ★★★★☆ | ★★★☆☆ | 高 | **高价值跟进** |
| 化工/危化/应急 | ★★★★★ | ★★★★☆ | ★★★☆☆ | 中高 | 重点拓展 |
| 工业/物流/农业 | ★★☆☆☆ | ★★★★★ | ★★★★★ | 低中 | **飞轮引擎** |
| 无人机+机械臂 | ★★★★☆ | ★★☆☆☆ | ★★★☆☆ | 很高 | 中期预研 |
| 深海/矿山/空间 | ★★★★★ | ★★★☆☆ | ★★☆☆☆ | 很高 | 长期机会 |

> **组合策略("两条腿走路")**:用**高危场景(电力/核/化工)**建立差异化壁垒与高毛利订单,同时用**工业/物流等高频场景**作为数据飞轮引擎快速积累自主能力 —— 后者滚出的通用操作技能会反哺前者,降低危险场景的自主化成本。

## 6. 结论与建议

1. **方案本身成立且主流**:四层"快慢思考"分层 VLA + 遥操作数据飞轮,方向与主流一致,硬件已为数据采集预留能力,Demo 已验证闭环。
2. **缺失的数据质量环是关键补齐项,且已在架构中被认知**:"任务完成度评价"已列在快思考层,只差落地。建议按第 4 章建设**帧级 + 回合级两级质检流水线**并做成独立"数据质量服务"。
3. **数据使用应分阶段**:短期(模仿学习练单点技能)→ 中期(VLA + 完成度自评 + 仿真协同压成本)→ 长期(云端规划蒸馏 + 长程自主)。
4. **场景上"高危滩头 + 高频引擎"两条腿**:以**高压带电作业**为首发滩头,核/化工跟进;工业/物流做飞轮引擎;**无人机+机械臂**作中期预研而非首发;**AR 眼镜遥操作**作为通用交互能力优先投入。
