## 01 · 第一性原理:为什么机器人非要一个"世界模型"?

报告开篇用一张图讲了本质逻辑,翻成大白话是四步:

1. **真实状态看不全** —— 世界有完整真实状态 `sₜ`,但谁也没法直接、完整看到它。
2. **只能看到片面** —— 传感器只给局部观测 `oₜ`(摄像头、本体感知、记忆、上下文拼出的片面画面),是对真实状态的一次"投影"。
3. **用动作去试探** —— 试走几步候选动作 `aₜ:ₜ₊H₋₁`,看模型能否预判后果。
4. **能预测=真懂了** —— 若模型能准确预测接下来的观测 `ôₜ₊₁:ₜ₊H`(帽子=预测值,H=规划视野 horizon),说明其内部的潜在信念 `zₜ`(latent,人看不见、藏在网络里的表示)已抓住控制所需的隐藏结构。

- ✓ 已核实:这套 `s / o / z / ô` 记号是**基于模型的强化学习(model-based RL)** 标准写法(PlaNet / Dreamer 一脉),准确无误。
- ℹ 提醒:它来自强化学习文献,并非源自后文的"李飞飞分类法",两者别混。

**第一性原理:世界模型 = 用"内部模拟"替代"昂贵的真实试错"。** 真实世界试错代价高、慢、可能摔坏;若脑中有够准的世界模型,就能在想象里把几种做法各推演一遍,选最靠谱的再动手。这是 2018 年 Ha & Schmidhuber《World Models》以来的目标——**在想象中规划(planning in imagination)**。报告的"数据飞轮"正是它的工程化。

## 02 · 行业地图:世界模型的三种"活法",以及它们正在收敛

借用**李飞飞 / World Labs 分类法**(一手源为 World Labs 官方博客《A Functional Taxonomy of World Models》,2026-06,以 POMDP 为底座),按"最终吐出什么"分三类:

- **渲染器 Renderer → 输出观测:** 把提示/动作/场景变成像素、视频、画面,追求"看着像真的"。代表:DeepMind Genie 2/3、OpenAI Sora。
- **模拟器 Simulator → 输出未来状态:** 预测几何/物理/环境在力和动作下怎么演化,追求"物理上说得通"。代表:Waymo World Model、NVIDIA Cosmos、Wayve GAIA-2。
- **规划器 Planner → 输出动作:** 用预测的未来,在目标/约束下选动作,追求"能干活"。代表:Fast-WAM、ScoutWAM(我方)。

**行业主线:三条线收敛到"世界-动作模型 WAM":**

- 2023:渲染器先行(海量网络视频→Genie、Sora、Runway Gen-4、阿里 Wan、腾讯 HunyuanWorld…)。
- 2024–25:模拟器跟上并结合空间数据,自动驾驶尤为集中(Waabi World、GAIA-2、Cosmos、Waymo WM)。
- 2025–26:规划器登场,收敛到 WAM——因为这里才是"感知与动作形成闭环(做了看结果、再指导下一步)"、也是商业价值落地处。

- ✓ 已核实:上述模型均真实存在、时间线基本吻合(Genie 3 为 2025-08;Waymo WM 为 2026-02 且基于 Genie 3)。
- ✕ 查无:报告时间线图里的 **WAV(World-Value-Action)** 与 **OA-WAM** 公开渠道搜不到,疑为内部/新造术语;对外展示建议标注或替换。

**根因(报告最扎实的一句话):动作数据才是瓶颈。** 数据丰富程度天差地别:

- **文本**:互联网规模、几乎免费,最多——但不含动作信号。
- **视频**:网络海量、便宜——但没有动作标签。
- **仿真**:可批量造——但需引擎+场景+验证,还有 sim-to-real 落差(仿真学的技能搬到真实会打折)。
- **真实动作**:要机器人、任务、复位、失败、安全、监督——最稀缺、最贵。

- ✓ 已核实:此"数据金字塔"是业界公认方向性共识("动作数据不存在于互联网上"被反复引用)。
- ℹ 提醒:属定性排序、非官方定量定义。报告由此推出的核心能力主张——"从一切对象(人/物/动物及交互)提取统一运动信号"——正是想把便宜的视频尽量榨成动作证据,绕开瓶颈。

## 03 · 立项逻辑:为什么是我们——这是"具身系统"问题,不只是"堆模型"问题

报告用巡检机器人闭环说明我方位置,并把现状/目标摆成对照:

**现状 AS-IS(痛点)**
- ① **遥操作无法规模化**:人远程操控机器人采数据(teleoperation),随任务花样变多,成本高到不可承受;每条示范绑定特定硬件/场地/物体,无法像抓网页那样批量获取。
- ② **缺少"反事实"**:遥操作只记录"人当时怎么做的",没记录机器人本该在行动前评估的其它可能未来(counterfactual="如果换个做法会怎样"),而选最优动作恰恰需要它。

**目标 TO-BE**
- ③ **运行时数据闭合循环**:WAM 让我们用机器人真实作业中产生的视频+控制轨迹,把数据飞轮转起来。
- ④ **自我改进变得可行**:WAM 让模型能感知自己的成功与失败,识别薄弱案例,从自身结果学习。

- ℹ 判断:这是报告里最经得起推敲的战略论证。核心论点站得住——下一步优势来自把"预测"接到"真实动作+高质量证据+闭环学习"上,而非单纯做大模型;我方同时握着**硬件具身入口、评估纪律、可部署产品场景(巡检)**,是纯模型团队不具备的位置。

## 04 · 我们的模型:ScoutWAM,一场"小而强"的赌注

**设计哲学:** 先找到模型的"认知核心",再把多余的砍掉(用内部业务标准与基准做消融=逐一去掉部件看性能变化)——保持小规模,让每个参数都算数。

**与两个对标的比较(报告口径):**

| 模型 | 参数量 | 思路 | 核实 |
|---|---|---|---|
| 星海图 Fast-WAM | 6B | 去掉渲染器、保留规划器,接近"认知核心" | ✓ 真实论文;6B ≈ Wan2.2-5B + 1B 动作专家,吻合 |
| **ScoutWAM(我方)** | **2.06B** | 把 LeWM(JEPA)的潜在规划 + 新的潜空间正则结合,2B 就匹配性能 | ✓ LeWM/JEPA 真实;ℹ 应讲清 LeWM 是 JEPA 的一个实现,而非等同 |
| NVIDIA Dream Zero | 14B | 报告称 ScoutWAM 远小于它 | ✕ 查无 公开渠道搜不到"Dream Zero";NVIDIA 世界模型线为 Cosmos。建议核实或替换 |

> 术语:**JEPA(联合嵌入预测架构,LeCun 提出)**=在潜空间预测未来的"表示",而不重建每个像素,更省、更抓本质。**LeWM = LeWorldModel**(arXiv 2603.19312,LeCun 等)是 JEPA 的一个具体实现。

**架构拆解(每个部件在干嘛):**

- **紧凑 Wan2.1 1.5B VAE 主干**:VAE=变分自编码器,把高清画面压成一小串潜变量再解码,大幅省算力;用更小的压缩路径,让"预测观测"在动作推理前就足够便宜。(✓ VAE/Wan2.1 真实,阿里开源,arXiv 2503.20314)
- **仅训练期使用的未来视频潜变量**:训练时学"未来怎么变",推理时不再费力生成未来视频,只聚焦"观测→动作"。(ℹ 与 Fast-WAM 论文思路一致)
- **直接动作 token 路径**:token=模型处理的最小信息单元;把动作编码成 token 让模型直接产出动作,保住"和策略的相关性",避免退化成只会预测视频的花架子。
- **SIGReg 稳定化**:报告标注的潜空间稳定化/正则化方法(JEPA 家族思路),防止想象轨迹跑飞,使其对下游控制仍可用。(精确定义建议以 JEPA/LeWM 相关论文为准)
- **潜在规划头**:执行前用想象的未来给候选动作打分/优化,选最好的再做。
- **目标:可部署规模**:2.06B 瞄准"能真正装进机器人"的体量,而非追更大基线体积。

**关键指标(团队自报):**

- **91.1%** —— LIBERO 四套件平均成功率(LIBERO=UT Austin 2023 机器人操作标准基准,含 Spatial/Object/Goal/Long 即 LIBERO-10 四套件)。
- **2.06B** —— 紧凑世界-动作模型规模。
- **52.0** —— 双 GPU 评估下每 GPU 小时 rollout 数(rollout=从头到尾跑完一整段任务轨迹)。
- **4.21×** —— 相较基线更高的 LIBERO-10 rollout 吞吐。
- 另:**2,000 个 episode 中 1,822 次成功**(=91.1%)。

**⚠ 最需要补强的三点(对外一定会被问):**

1. **91.1% 是"标准协议"分,不代表鲁棒性。** 近期工作(如 LIBERO-PRO)指出,加入位置扰动等更公平设置后,这类头部分数会明显下降。须注明评测协议。
2. **"2B 匹配 6B/14B"要有并列对比表。** 匹配的是哪些指标?同基准、同协议吗?否则"同样强大"只是口号。
3. **"4.21×""52 rollouts/GPU-hour"的基线是谁、硬件是否一致?** 吞吐优势很可能部分来自"推理期不生成未来视频"——这本身是亮点,但要说清对比条件。

## 05 · 核心机制:数据飞轮——让"失败"自动变成下一轮教材

**关键不是"跑更多 rollout",而是让模型比较"它想象的"和"它真实看到的",找出想象不靠谱之处,专门为薄弱案例生成新训练数据。**

闭环步骤:

1. **状态信念**(观测+记忆+目标约束)→
2. **候选动作** a₁ … a_K →
3. **世界模型:想象** —— 为每个候选动作预测未来观测 ô →
4. **成功评分器** —— 按目标/安全/风险排序,选 `a* = argmax_a V(ô, g)`(在所有候选里挑"想象未来相对目标 g 得分 V 最高"的动作)→
5. **执行** —— 做选定动作,观测真实结果 o →
6. **差异估计器** —— 比较想象 vs 真实:`Δ = d(ô, o)`(距离;Δ 越大越"想错了")→
7. **定向数据** —— 在 Δ 高的场景,生成更多想象 rollout 作纠错教材 →
8. **闭环** —— 想象→行动→观测→纠错→改进下一轮。

**ℹ 第一性原理层面,飞轮的"阿喀琉斯之踵"(最该正视的深度问题):**

- **复合误差**:世界模型想象越远,误差越滚越大;用"想象数据"训练自己,可能把偏差越练越大(确认偏误 / 模型坍缩风险——反复用自生成数据训练导致分布越来越窄、越失真)。
- **"想象不可靠的地方",恰恰也是"Δ 本身不可靠的地方"**:模型在陌生场景既想不准、也不知道自己想不准。所以 Δ 必须用**真实世界的 o 来锚定**,不能闭门自训。
- **推论:里程碑二(真机 R1-Lite)不是可选项,而是飞轮成立的前提**——没有真实的 o 校准想象的 ô,"自我感知"会退化成"自我欺骗"。这一点值得在汇报里显式点出,反而更显功力。

**里程碑:**

- **M1**:引入想象空间 rollout,实现闭环仿真(想象→行动→观测→度量差异→改进),打开自学习路径。
- **M2**:把 ScoutWAM 部署到具身 R1-Lite 真机平台(星海图 2026-01 发布的机器人平台),从"基准证明"迈向"真实部署闭环",攻更难的语义任务。
- **M3**:把真机失败/险些失败/未完成任务按类型(感知/接触/物体意图/空间关系/完成度/安全)标注,转成定向纠错数据微调下一版。

## 07 · 术语表(一次看懂)

- **世界模型 / World Model**:AI 内部对"世界如何运转"的可预测模型,能推演"做了某动作后会发生什么"。
- **具身智能 / Embodied AI**:有身体(机器人)、能在物理世界感知与行动的 AI。
- **POMDP**:部分可观测马尔可夫决策过程;真实状态看不全,只能靠局部观测推断决策。世界模型的标准数学框架。
- **潜空间 / 潜变量 / 潜在信念**:模型内部压缩出的低维表示(latent),人看不见但抓住关键结构;zₜ 即当前潜在信念。
- **渲染器 / 模拟器 / 规划器**:世界模型三分法,分别输出"观测 / 未来状态 / 动作"。
- **JEPA**:联合嵌入预测架构(LeCun);在潜空间预测未来"表示",不重建像素。LeWM 是其一个实现。
- **VAE / Wan2.1**:变分自编码器,把画面压成潜变量再解码,省算力;Wan2.1 是阿里开源视频模型,提供 VAE 主干。
- **VLA vs WAM**:VLA=视觉-语言-动作,直接"看→做"(反应式);WAM=世界-动作模型,多一层"先预测世界再行动"(前瞻式)。
- **rollout**:从头到尾跑完一整段任务轨迹,用于评测或采数据。
- **LIBERO**:机器人操作标准基准(UT Austin,2023),含 Spatial/Object/Goal/Long(LIBERO-10)四套件。
- **遥操作 / Teleoperation**:人远程操控机器人采集示范数据,精确但贵、慢、难规模化。
- **反事实 / Counterfactual**:"换个做法会怎样"的假想结果,是选最优动作所必需但遥操作没记录的信息。
- **消融 / Ablation**:逐一去掉模型某部件看性能变化,判断哪些是真正必要的"认知核心"。
- **SIGReg**:报告标注的潜空间稳定化/正则化方法(JEPA 家族思路),防止想象轨迹跑飞。
- **数据飞轮**:用运行中产生的数据(尤其失败)持续反哺训练、自我改进的闭环。
- **复合误差 / 模型坍缩**:预测越远误差越累积;反复用自生成数据训练会越练越窄、越失真。飞轮的核心风险。
