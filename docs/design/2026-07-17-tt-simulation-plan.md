## 2. 关键概念:数字孪生 ≠ 仿真环境

这是理解"仿真怎么串起来"的核心。两者数据流方向相反:

### 当前:数字孪生(已有)

真机是真身,MuJoCo 是镜子。用于可视化与轨迹记录。

```
真无人机飞行
   │  Tello SDK 上报位姿
   ▼
Mission Pad 相对坐标 x/y/z/yaw
   │  照抄
   ▼
MuJoCo 虚拟机同步 + 画轨迹
```

> ⚠️ 必须有真机在飞才有画面 · 无法开发避障

### 目标:闭环仿真(要建)

仿真是真身,虚拟机自己飞。用于安全开发与测试避障。

```
┌─────────────────────────────────────────────┐
│  虚拟无人机(在含障碍的虚拟世界)               │
│     │  机载相机(EGL 渲染图像)                 │
│     ▼                                         │
│  视觉避障算法 ── 输出速度指令 ──┐              │
│     ▲                          │              │
│     └────── 驱动虚拟机移动、遇障躲避 ◀────────┘ │
└─────────────────────────────────────────────┘
```

> ✅ 不用真机 · 可反复跑上千次 · 安全迭代

## 3. 三种仿真构建路线对比

| 方案 | 怎么做 | 优点 | 缺点 | 适用 |
|------|--------|------|------|------|
| **① 纯仿真** (sim-first) | MuJoCo 里加障碍物 + 给虚拟机装前视相机,渲染图像喂算法,算法输出指令驱动虚拟机闭环 | 🟢 安全:不用真机、可反复跑上千次、迭代快;4090+EGL 现成 | 🔴 sim-to-real:真 Tello 相机畸变/延迟/控制响应有差距 | 开发+调试避障 |
| **② 真机+孪生** (现状) | 真 Tello 飞,MuJoCo 做镜子记录轨迹 | 真实数据、无 sim-to-real 差距 | 🔴 受限:必须有真机和场地、会炸机、无法离线开发、Tello 传感有限 | 最终真机验证 |
| **③ 混合** ⭐**推荐** | 先在 ① 里把避障做出来调好,再用 ② 上真机验证落地 | 🟢 兼顾:安全快速迭代 + 真实验证;现有孪生正好当"真机验证器",不浪费 | 两边都要建、需处理 sim-to-real 对齐 | 完整开发闭环 |

---

## 4. 推荐:混合 ③,从 ① 起步

- 避障算法炸机代价高 —— **必须先在仿真里练熟**,不能拿真机试错。
- 本机 **4090 + EGL 已验证**,做视觉闭环成本低,渲染快。
- 现有孪生天然就是 ③ 里"真机验证"那一环 —— **不浪费已有工作**。
- 与大众用 `wam` 做自动导航的思路一致:**导航大脑先在仿真里养熟,再上真机**。
